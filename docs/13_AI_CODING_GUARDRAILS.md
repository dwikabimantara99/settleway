# 13 - AI Coding Guardrails

## Do not do these unless approved

- Change product name.
- Change stack.
- Add real payment integration.
- Add real KYC.
- Add complex chat.
- Add insurance.
- Add logistics marketplace.
- Add multi-sector support.
- Replace Stellar with another blockchain.
- Build a full AI dispute judge.
- Rewrite existing completed phases without approval.

## Required response after each phase

Return:

1. What was implemented.
2. Files created.
3. Files modified.
4. Commands run.
5. How to test.
6. Known limitations.
7. Whether acceptance criteria are met.
8. Stop and wait.

## If ambiguity appears

Ask. Do not invent product direction.

## If build fails

Report exact error, likely cause, proposed fix, and ask before large refactors.
